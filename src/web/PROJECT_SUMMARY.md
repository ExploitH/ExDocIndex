## 💡 核心创新点

1. **文件状态机设计**
   - uploaded → understanding → understood → indexed
   - 每个状态都有严格的转换规则

2. **强关联约束**
   - 源文件删除 → 自动级联删除理解文件和索引
   - 通过数据库外键和文件状态管理器实现

3. **单线程任务队列**
   - 防止并发操作导致内存爆炸
   - 任务顺序执行，稳定可靠

4. **双模式索引**
   - 二次索引：基于理解后的文件（高质量）
   - 直接索引：基于原始文件（需警告确认）

5. **错误日志系统**
   - 自动保存错误详情到文件
   - 包含任务信息、堆栈跟踪、时间戳
